# Rate Limit 视觉标识设计分析

## 设计目标

当 Codex Monitor 中某个账户达到使用限额时，需要一个明确的视觉标识，让用户一眼看出哪个账户不可用。

## 三个方案对比

### 方案 A — 红色遮罩 + LIMITED 徽章 + 删除线
**文件**: `variant-a-overlay-stamp.html`

| 维度 | 评价 |
|------|------|
| 视觉冲击 | ⭐⭐⭐⭐⭐ 最强，红色 + 徽章 + 删除线三重信号 |
| 信息密度 | ⭐⭐⭐ 保留了原始数据但被遮罩降低可读性 |
| 实现复杂度 | ⭐⭐ 中等，需要 overlay + 徽章 + 线条动画 |
| 空间占用 | ⭐⭐⭐⭐⭐ 不额外占空间 |

优点：辨识度最高，LIMITED 印章效果有品牌感，不改变布局结构
缺点：锁定图标遮罩可能让数据不可读，信息传达偏"情绪"而非"实用"

### 方案 B — 限额已达横幅 + 暗化卡片
**文件**: `variant-b-pill-banner.html`

| 维度 | 评价 |
|------|------|
| 视觉冲击 | ⭐⭐⭐⭐ 强，红色横幅 + 脉冲动画 |
| 信息密度 | ⭐⭐⭐⭐⭐ 最高，数据仍在 + 状态说明 + 重置时间 |
| 实现复杂度 | ⭐⭐⭐ 简单，横幅 + 暗化样式 |
| 空间占用 | ⭐⭐⭐ 增加一行横幅高度（约 32px） |

优点：信息最完整，带重置时间预估，数据仍可见，实现最简单
缺点：多一行横幅空间，不如方案 C 视觉独特

### 方案 C — 卡片替换为限额状态面板
**文件**: `variant-c-card-replace.html`

| 维度 | 评价 |
|------|------|
| 视觉冲击 | ⭐⭐⭐⭐ 强，环形仪表盘 + 锁定图标 |
| 信息密度 | ⭐⭐⭐⭐ 高，状态 + 详情 + 倒计时 |
| 实现复杂度 | ⭐⭐ 最复杂，需要 SVG 仪表盘 + 倒计时逻辑 |
| 空间占用 | ⭐⭐⭐⭐⭐ 与原双卡片等高 |

优点：信息架构最清晰，环形仪表盘视觉独特，倒计时有期待感
缺点：实现复杂度最高，需要动态计算重置时间，SVG 渲染成本

## 推荐方案

**推荐：方案 B（限额横幅 + 暗化卡片）**

理由：
1. **实现最简单** — 纯 CSS + 少量 JS，不需 SVG 或复杂动画
2. **信息最完整** — 同时传达"限额已达"状态、哪些限额用尽、何时重置
3. **向后兼容** — 卡片结构不变，仅增加横幅和样式类
4. **可渐进增强** — 先做静态横幅，后加倒计时功能

如果 Henry 偏好更强的视觉冲击，方案 A 的 LIMITED 徽章可以作为方案 B 的补充元素加入。
